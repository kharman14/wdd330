Master Mobile UX

Start with the smallest device and work up. Just because your website is responsive doesn't mean that it's mobile first. Start small and scale up. Think about user expereince, the less steps for the users the better. Think of how people are using their devices. Keep network and loading times in mind: SVG/Webfonts are lower bandwidth (aka load faster). Optimize and minify CSS and JS files. Moving the menu to the bottom of the screen helped with reachability but also perceived speed. Mobile is still very new, so there might be better ways to do things than how they're currently being done. Make it simply. Reduce cognitive load.
